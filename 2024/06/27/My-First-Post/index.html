使用 Hexo 项目并在 GitHub 上部署指南

Hexo 简介

Hexo 是一个快速、简洁且高效的博客框架，基于 Node.js 开发，适用于个人和团队博客的搭建。

我为什么选择Hexo,作为博客框架的搭建

原因：近期在使用air2软件的服务中，相关bt离线服务无法正常下载，在google搜索后发现了一个对该问题分析原因的网站[网站地址]（https://p3terx.com/ “p3terx”），点了点，加载响应超级快，体验感觉非常好，我就希望我的博客主站就用这个来做吧，那么是怎么做呢，先通过openai的chatgpt 进行交互，了解到网站是由Hexo 及butterfly主题来生成的。那么我决定来搭建一个属于个人的博客网站

步骤一：安装 Hexo

  1. 前置：本地服务一定有 Node.js 和 npm

    • 访问 Node.js 官网 下载并安装最新版本的 Node.js。
    • npm 会随 Node.js 一同安装。

  2. 安装 Hexo

    • npm install -g hexo-cli
    • hexo init myblog

  3. hexo的一些指令及文件格式介绍

步骤二(两种方式)：配置 Git 与 GitHub

- 在 GitHub 上创建仓库
- 登录 GitHub，创建一个新的空仓库，例如 username.github.io（其中 username 是你的 GitHub 用户名）。
- 配置 Git
- 初始化 Git 仓库并关联远程仓库：
    - git init
    - git remote add origin https://github.com/username/username.github.io.git

编写文章与提交更改

使用 Hexo 创建新文章：
bash
复制代码
hexo new “Hello World”
编辑文章并保存。
生成静态文件

bash
复制代码
hexo generate
部署到 GitHub Pages

安装 Hexo 部署插件：
bash
复制代码
npm install hexo-deployer-git –save
配置 _config.yml 文件：
yaml
复制代码
deploy:
type: git
repo: https://github.com/username/username.github.io.git
branch: master
部署到 GitHub：
bash
复制代码
hexo deploy
步骤三：Git 基本操作
查看文件状态

bash
复制代码
git status
添加文件到暂存区

bash
复制代码
git add .
提交更改到本地仓库

bash
复制代码
git commit -m “提交说明”
推送本地更改到远程仓库

bash
复制代码
git push origin master
步骤四：Hexo 常用命令
启动本地服务器

bash
复制代码
hexo server
清除缓存文件

bash
复制代码
hexo clean
生成静态文件

bash
复制代码
hexo generate
步骤五：Hexo 主题与配置
安装 Hexo 主题

在 GitHub 上找到喜欢的 Hexo 主题，例如：
bash
复制代码
git clone https://github.com/theme-author/theme-name.git themes/theme-name
配置主题

修改 _config.yml 文件，指定使用的主题：
yaml
复制代码
theme: theme-name
自定义配置

根据主题文档修改 _config.yml 文件，配置个性化内容。
